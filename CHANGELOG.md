# 更新日志

本项目遵循语义化版本，记录 fork 后的主要变更。

## 未发布

### 改进

- CI 分析流程改为使用锁文件获取依赖，并在分析前运行完整测试。
- SQLite 后台 isolate 操作改为独立打开数据库连接，减少跨 isolate 复用连接带来的稳定性风险。
- 数据导入流程增加替换前备份与失败回滚，并等待相关数据库管理器重新初始化完成。
- 应用数据保存改为排队写入，避免并发保存时的忙等和互相抢占。
- WebDAV 数据同步改为显式任务排队，减少上传/下载并发等待时的轮询。
- 漫画源数据保存改为合并排队写入，减少设置频繁变更时的重复等待。
- 文件下载器改用写入队列串行落盘，减少多块并发下载时的忙等和缓冲累积。
- JS 计算池初始化改为共享 Future，并用完成通知替代 isolate 任务关闭轮询。

### 修复

- 修复 `Channel` 多个生产者等待同一释放信号时可能突破容量限制的问题。
- 修复文件下载分块请求失败时任务清理和错误传播不稳定的问题，并补充异常路径测试。
- 修复 WebDAV 数据同步失败结果未统一刷新错误状态的问题，并补充异常路径测试。
- 修复阅读器快速退出时最后阅读进度可能未写入的问题。
- 修复 JS 引擎重置时旧实例未正确释放的问题。
- 修复网络内存缓存可能超过上限的问题，并补充缓存回归测试。
- 替换 Flutter 已弃用的 `TickerMode.of` API。

## v1.9.0

### 新增

- WebDAV 数据同步新增状态展示，可显示同步中、上次同步时间和失败信息。
- 新增历史记录自动清理设置，支持按保留天数清理过期历史记录。
- 新增本地漫画导入导出工具层，为后续本地漫画迁移入口提供基础能力。

### 改进

- 统一主要 SQLite 数据库连接配置，设置 journal mode、同步策略和 busy timeout。
- WebDAV 数据同步导入后会刷新历史、收藏和图片收藏状态，减少首页与收藏计数滞后。
- 完善 WebDAV 漫画归档连接测试和错误提示。

## v1.8.1

### 新增

- 关于页面新增更新日志入口，可直接查看 `CHANGELOG.md` 内容。

### 改进

- App 版本号改为从打包内 `pubspec.yaml` 读取，发版时只需维护 `pubspec.yaml`。
- 检查更新改为 CDN 版本信息优先、GitHub 最新发行版兜底，并在失败时提示用户。

## v1.8.0

### 新增

- 新增 WebDAV 本地漫画归档功能，支持备份、恢复、列出和删除本地漫画归档。
- 新增图片收藏图库视图，便于集中浏览和管理单部漫画的图片收藏。
- 新增手动构建工作流，支持按平台选择构建任务。

### 改进

- 增强 tag 自动发布流程，发布前校验版本号、更新日志和关键依赖锁定状态。
- 强化 `flutter_rust_bridge 2.11.1` 锁定检查，降低构建产物无法联网的风险。
- 改进 CBZ 导出兼容性，统一页码文件名、章节范围和 `ComicInfo.xml` 元数据。
- 改进导出文件名处理，增强多语言标题、章节后缀和 UTF-8 长度限制兼容性。
- 简化 README 中的项目说明和构建提示。

### 修复

- 修复发布工作流中 iOS 打包路径硬编码旧仓库名的问题。
- 修复发布工作流中 Linux x64 构建未显式使用锁文件获取依赖的问题。
- 移除关于页面中残留的 Telegram 入口。

## v1.7.0

### 新增

- 新增瀑布流阅读模式，作为连续阅读模式的跨章节增强。
- 瀑布流模式复用现有“预加载图片数量”，在当前位置后续图片不足时自动加载下一章。
- 瀑布流阅读时会根据当前可见图片同步章节和阅读进度。

### 修复

- 修复瀑布流阅读模式名称缺少简繁中文翻译的问题。
- 瀑布流跨章节处增加章节分隔提示，便于识别章节边界。
- 修复瀑布流跨章节后无法向上滚回上一章的问题。
