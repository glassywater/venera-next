# 更新日志

本项目遵循语义化版本，记录 fork 后的主要变更。

## v1.11.0

### 新增

- 为纵向连续和瀑布流阅读模式增加拆分双页功能，可将横向双页图按阅读顺序拆成上下排列，并支持交换拆分顺序。
- 本地漫画 CBZ/ZIP/7Z 导入支持顶层目录下的章节子目录结构，便于导入整部漫画打包后的单文件归档。
- 新增 WebDAV 漫画库在线阅读渠道，仅支持远端目录图片结构按需加载；CBZ/ZIP/7Z 继续作为导入导出、归档和分发格式，不用于在线预览。

### 修复

- 修复瀑布流和纵向连续阅读到章节末尾时，阅读记录可能仍显示剩余 1-2 张图片未读的问题。
- 修复多个 toast 连续出现时位置重叠，导致下方提示被遮挡的问题。
- 修复部分 CBZ/ZIP 使用非 UTF-8 文件名编码时，本地导入可能抛出 `FormatException: Missing extension byte` 的问题。
- 修复部分 Windows 压缩得到的 CBZ/ZIP 在原生解压器和 7z 均失败时，导入提示 `Failed to open archive` 的问题。
- 修复 WebDAV 漫画库读取中文目录或不可变目录列表时，可能显示 `unsupported operation: cannot remove from an unmodifiable list`，以及详情页章节长时间加载的问题。

### 文档

- 按文档类型整理 `doc/` 分类目录，补充中英文文档索引、漫画源开发中文说明和图片增强实验设计记录。
- 扩充 README 使用说明，细化本分支特色、本地漫画导入、CBZ 分发归档、WebDAV 在线漫画库和 WebDAV 数据同步/归档配置区别。
- 明确区分本地漫画压缩包导入导出、WebDAV CBZ 归档和 WebDAV 漫画库在线阅读三类能力。

## v1.10.0

### 新增

- 新增漫画更新提示开关，默认在缩略图角标和漫画详情页提示自上次阅读后的未读更新。
- 新增 `release.json` 作为发布版本号的统一来源，并提供脚本同步校验 `pubspec.yaml`、发布 tag 与更新日志版本章节。

### 变更

- 长按漫画详情页收藏按钮默认快速添加到「追更」收藏夹，同时保留用户设置的有效快捷收藏夹。
- 首页追更栏显示「追更」收藏夹内的漫画封面预览，更新数量仅作为附加提示。
- 漫画详情页阅读记录显示当前章节进度，便于判断已读到总章节中的位置。
- 将各平台发布者、版权、应用 ID 与 AltStore 源身份迁移到 `CyrilPeng/venera-next`；这是破坏性升级，系统会将其视为新的应用身份。
- 将应用展示名、桌面可执行文件、安装目录和发布产物统一命名为 `VeneraNext`；Linux 命令和包名使用 `venera-next`。
- 漫画详情页桌面端与移动端统一为两行按钮布局，阅读统一通过主按钮或章节列表进入。
- 为漫画详情页第二行主按钮补充下载、阅读和继续阅读图标。
- Windows 发布构建会清理旧 runner 输出、声明图标资源依赖并校验图标资源，避免桌面包继续沿用旧图标缓存。
- RC 预发布的 iOS 构建改为使用 Apple 平台兼容的数字短版本，同时保留外部产物文件名中的 RC 版本号。
- Windows 应用元数据中的公司名改为反向域名 `com.github.cyrilpeng`，使个人数据目录使用更规范的 `com.github.cyrilpeng\VeneraNext` 结构，并在新目录为空时自动迁移旧目录数据。
- 缩短 Windows 和 macOS 发行版版权信息，避免属性面板中显示不全。
- 长按漫画详情页收藏按钮后的提示会显示目标收藏夹，避免“已添加”表述不清。

### 修复

- 修复首次启动自动创建「追更」收藏夹后未立即设为默认追更文件夹，导致本轮启动不检查追更的问题。
- 修复删除、移动或重命名追更收藏夹内容后更新角标缓存未及时刷新，导致缩略图更新提示残留的问题。
- 修复正式版用户会收到 RC 预发布版本更新提示的问题，预发布更新仅提示当前使用预发布版本的用户。
- 修复漫画详情页浏览量等只读信息被误作为可点击搜索标签的问题。
- 修复 Windows 便携 ZIP 解压后文件直接散落的问题，改为包含同名顶层目录。
- 修复 Windows ARM64 便携 ZIP 错误使用 x64 构建目录的问题。
- 修复 Arch Linux 预发布包仍按旧可执行文件名 `venera` 打包，导致 `venera-next` 桌面构建失败的问题。
- 修复版本同步脚本匹配空白过宽，避免同步发布元数据时吞掉空行或改写文件换行。
- 修复 Windows 桌面端通过 WebDAV 恢复数据后，旧网络请求实例继续使用已关闭的 Cookie 数据库连接，导致漫画详情页全部加载失败的问题。
- 修复长按追更触发首页历史和本地预览刷新时，封面短暂变成黑色的问题。

## v1.9.1-rc.1

### 新增

- 发布校验支持 `v1.9.1-rc.1` 形式的 RC 标签，并在 GitHub Release 中自动标记为预发布。

### 变更

- CI 分析流程改为使用锁文件获取依赖，并在分析前运行完整测试。
- SQLite 后台 isolate 操作改为独立打开数据库连接，减少跨 isolate 复用连接带来的稳定性风险。
- 数据导入流程增加替换前备份与失败回滚，并等待相关数据库管理器重新初始化完成。
- 应用数据保存改为排队写入，避免并发保存时的忙等和互相抢占。
- 历史记录异步写入改为 Future 队列，减少阅读进度频繁保存时的轮询等待。
- 历史记录批量刷新改为共享节流并发调度器，简化固定 worker 队列并保留批次节流。
- 关注更新批量检查复用节流并发调度器，减少固定 Channel worker 队列的重复实现。
- 共享节流并发调度器在最后一批任务后不再等待额外节流间隔。
- 收藏批量移动/复制改为事务提交后再通知，并在全文件夹删除提交后刷新计数。
- 收藏空批量移动、复制和删除会提前返回，避免无意义事务和监听通知。
- 本地漫画单项删除收敛为一次本地管理器通知，减少重复刷新。
- WebDAV 数据同步改为显式任务排队，减少上传/下载并发等待时的轮询。
- WebDAV 同步任务等待移除完成后的空延迟，减少排队任务切换时的无意义调度。
- 关闭窗口等待 WebDAV 上传时改为等待当前上传任务，减少退出前的固定间隔轮询。
- 关注更新后台检查改为等待 WebDAV 下载任务完成，避免固定间隔轮询和排队下载竞态。
- 漫画源数据保存改为合并排队写入，减少设置频繁变更时的重复等待。
- 漫画源设置解析与动态读取共用归一化逻辑，并统一包装 JS 回调设置项。
- 漫画源图片加载配置统一标准化，兼容 JS 返回的动态 Map 并拒绝非字符串键。
- 漫画源详情、章节图片和缩略图入口统一归一化 JS 返回值，减少动态 Map/List 类型边界问题。
- 漫画源详情入口补充嵌套字段归一化，兼容 JS 返回的动态推荐、评论和标签数据。
- 漫画源评论和章节评论入口统一归一化 JS 返回值，减少动态评论列表类型边界问题。
- 漫画源归档列表和归档下载地址入口统一校验 JS 返回值，减少归档下载类型边界问题。
- 漫画源搜索、分类和收藏列表入口统一归一化漫画条目，减少动态 Map 类型边界问题。
- 文件下载器改用写入队列串行落盘，减少多块并发下载时的忙等和缓冲累积。
- 缩略图加载并发限制改为显式许可队列，减少高并发加载时的固定间隔轮询。
- 图片响应回调改为统一执行并在异常路径释放，避免 JS 回调引用泄漏。
- 图片加载失败回调改为捕获后统一消费并释放，减少重试配置切换时的引用残留风险。
- 图片加载失败回调兼容 JS Map 返回的动态配置，并拒绝非字符串键配置。
- 图片加载失败重试按剩余预算判断，避免失败回调比预期多执行一次。
- 图片加载重试等待支持取消信号唤醒，减少取消后仍等待重试间隔的停顿。
- 文件下载在任务创建阶段被停止时会关闭输出流和文件句柄，避免调用方等待挂起。
- 文件下载分块失败时会取消已启动分块并保留首个错误，避免错误路径被在途请求阻塞。
- 文件下载断点状态不足时不再继续发送完成状态，避免失败结果被误报。
- 文件下载进度定时器会在失败或提前结束时立即取消，避免关闭流后继续发送进度。
- 图片下载任务暂停或取消时会唤醒重试等待，减少退避延迟造成的暂停卡顿。
- 图片下载任务取消时会停止速度计时器，并在尚未创建下载路径时跳过本地库查询。
- 阅读器自定义图片处理等待改为取消信号驱动，减少固定间隔轮询并传播异步错误。
- 图片脚本处理并发控制改为显式许可队列，减少高频处理时的轮询等待。
- JS 计算池初始化改为共享 Future，并用完成通知替代 isolate 任务关闭轮询。
- 更新日志页面改为按 Markdown 样式渲染，提升标题、列表和行内代码的阅读体验。
- 替换 Flutter 已弃用的 `TickerMode.of` API。
